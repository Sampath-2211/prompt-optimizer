# 🔬 PromptLab

A meta-prompting tool that takes your vague idea and turns it into a well-structured, optimized prompt — then generates 4 technique-specific variations using independent LLM calls.

**Tech:** Python · Streamlit · LangChain · Groq (LLaMA 3.3 70B) · Sentence Transformers

## Key Philosophy

**LLMs are used ONLY where judgment is required** (interpreting intent, generating natural language). Quality scoring uses a **dual system: deterministic rule-based analysis + LLM judge** — disagreements are flagged, just like citation validation in production NLP systems.

## What It Does

Most people write vague prompts like *"Fix this code"* or *"Write a blog about AI"*. PromptLab acts as an AI prompt engineer — it interprets what you actually want, asks clarifying questions, and generates a structured prompt you can use in any LLM.

It generates **prompts, not solutions**. You take the output and run it wherever you want.

## How It Works

```
User Input → Interpretation → Clarification MCQs → Refined Understanding → Optimized Prompt + 4 Variations
```

1. You describe your task in plain English (optionally upload a file).
2. The system interprets your intent — identifies task type, objectives, expected output.
3. If the interpretation is off, you can either answer MCQ-style clarification questions or toggle to rephrase your idea entirely.
4. It generates an optimized prompt using the ARMS framework (Analysis, Reasoning, Methodology, Structure).
5. It generates **4 independent prompt variations** — each using a different prompting technique via separate LLM calls.

## Key Innovations

### 1. Independent Technique-Specific Prompt Generation

Each variation is generated by its own dedicated function with its own LLM call — not extracted from a single response. This ensures each technique is properly applied:

| Variation | Technique | What It Does |
|-----------|-----------|--------------|
| Chain-of-Thought | CoT | Breaks task into numbered reasoning phases with verification checkpoints |
| Few-Shot Guided | Few-Shot | Includes 3 realistic example scenarios demonstrating the expected approach |
| Role-Based Expert | Role-Based | Assigns a domain expert persona with credentials, methodology, and standards |
| Specification-Driven | Spec-Driven | Structures the prompt as a requirements doc with acceptance criteria and edge cases |

### 2. Dual Quality Scoring (Deterministic + LLM)

Two independent scorers evaluate your prompt:

- **Deterministic Scorer** — Rule-based analysis: counts structural signals (role assignment, output format, constraints, chain-of-thought triggers, examples, success criteria), measures vague word ratio, action verb density, sentence complexity. Instant, reproducible, zero LLM cost.
- **LLM Judge** — Nuanced assessment of clarity, specificity, structure, and completeness.
- **Grounding Agreement** — When the two scorers disagree by >20 points on any dimension, it's flagged. The LLM's judgment must be supported by structural evidence.

### 3. Adaptive Clarification

Users can toggle between two modes at any stage:
- **Refine Mode** — Answer MCQ questions + add free-text comments
- **Rephrase Mode** — Ignore questions entirely, describe the idea from scratch

## LLM vs Deterministic Split

| Task | Method | Why |
|------|--------|-----|
| Interpret user intent | LLM | Requires understanding unstructured text |
| Generate clarification questions | LLM | Requires contextual reasoning |
| Generate optimized prompt | LLM | Creative language generation |
| Generate CoT variation | LLM (independent call) | Technique-specific restructuring |
| Generate Few-Shot variation | LLM (independent call) | Example scenario generation |
| Generate Role-Based variation | LLM (independent call) | Expert persona creation |
| Generate Spec-Driven variation | LLM (independent call) | Requirements structuring |
| Score prompt (structural) | Deterministic | Pattern matching, regex, word counting |
| Score prompt (nuanced) | LLM | Subjective quality judgment |
| Grounding agreement check | Deterministic | Score comparison, threshold check |

## Setup

```bash
git clone https://github.com/Sampath-2211/PromptLab.git
cd PromptLab
pip install -r requirements.txt
```

Create a `.env` file:
```env
GROQ_API_KEY=gsk_your_key_here
```

Run:
```bash
streamlit run app.py
```
